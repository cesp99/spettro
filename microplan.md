# Project Specification: Spettro CLI

## 1. Objective
Build a **super-fast Go-based CLI tool** designed for coding assistance (similar to Open Code, Claude Code, or Cursor). The primary focus is efficiency and token optimization to accommodate users with API limits.

*   **Performance:** Must be extremely fast.
*   **Token Usage:** Minimize tokens per request. **Strict limit: < 10,000 tokens per request.** (Total session cost is secondary to per-request efficiency).
*   **Target Audience:** Users requiring multi-provider support due to API key limits or specific model preferences.

## 2. Architecture & Agents
The tool must operate with a dual-agent system plus a chat mode:

### A. Planning Agent
*   **Role:** Plans all actions before execution.
*   **Workflow:** Generates a plan -> Asks user for approval ("Build project" or "Make edits") -> Proceeds upon confirmation.
*   **Output:** All plans and indexing must be saved locally in the project repository (see Storage).

### B. Coding Agent
*   **Role:** Executes code changes based on the approved plan.
*   **Permission Levels:**
    1.  **YOLO Mode:** Full access to everything.
    2.  **Restricted Access:** Limited permissions for smaller tasks (e.g., creating folders, writing files).
    3.  **Standard/Ask-First:** Asks for permission before executing risky or high-level actions.

### C. Chat Agent
*   **Role:** General conversation mode between the Planning and Coding agents.

## 3. API & Provider Support
Support multiple providers to ensure flexibility. Use the following libraries:

1.  **OpenAI Compatible (Grok, Z.ai/GLM, Qwen, OpenRouter, OpenAi (ChatGPT):**
    *   Library: `github.com/openai/openai-go/v3`
    *   Note: These endpoints are compatible with this SDK.
2.  **Anthropic (Claude):**
    *   Library: `github.com/anthropics/anthropic-sdk-go`

## 4. User Interface & Interaction
*   **Visuals:** Implement a polished CLI UI. Use the file `eyes.txt` as a reference for the two main visual states (Planning vs. Coding). Ensure distinct themes/styles for each state.
*   **Navigation:**
    *   **Ctrl+Tab:** Switch between Planning Agent, Coding Agent, and Chat Agent modes.
    *   **/models Command:** Type `/models` to switch models or connect a new API key. The tool must save the selected model and key securely.
*   **Vision Support:** Allow users to paste images into the text field for vision-capable models.

## 5. Storage & Configuration
Distinguish between project-specific data and user configuration:

1.  **Project Context (Local Repo):**
    *   All files generated by agents (plans, indexing, context) must be stored in a `.spettro` folder within the **user's current repository**.
    *   Example Files: `PLAN.md`, `AGENT.md`.
2.  **User Configuration (Global Home):**
    *   User data (API keys, main agent settings applied to all sessions) must be stored in the user's home directory.
    *   Path: `~/.spettro`

## 6. Reference Materials
*   **Inspiration:** A folder `/insp` containing relevant repositories has been provided for code structure and feature inspiration.
*   **UI States:** The file `eyes.txt` contains the definitions for the UI states/themes.
